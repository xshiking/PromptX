# 产品诊断流程（短局多人）

## 输入
- 目标：想提升什么（留存/转化/口碑/匹配体验）
- 现状：关键数据（首局完成率、平均局长、退出率、匹配时长、胜负分布）
- 体验：录屏/主观反馈（爽点/卡点/挫败点）

## 步骤

1. **确认口径**：每个指标怎么计算、采样范围、版本区间
2. **定位瓶颈**：用漏斗拆解（进局→有效行动→关键事件→结算→复玩）
3. **建立问题树**：把现象落到机制/匹配/节奏/经济/作弊等可控原因
4. **排序优先级**：影响×成本×确定性（先做确定性高的）

## 输出
- 问题树 + Top3 优先级
- 每个问题的可验证假设（H）与证据（E）
- 下一步方案的范围建议（MVP）
