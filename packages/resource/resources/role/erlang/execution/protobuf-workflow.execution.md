# Protobuf 合同变更流程（含兼容与灰度）

## 步骤

1. **定义变更目标**：新增/修改哪个 module/method 的 payload
2. **更新 .proto**：新增字段优先；禁止复用字段号
3. **代码生成**：rebar3 生成/更新 pb 模块（保持生成路径一致）
4. **映射更新**：module/method → message 映射表（单一真相源）
5. **兼容检查**：旧客户端缺省字段、服务端校验、边界值
6. **联调与回归**：C2S/S2C 字节布局一致、压缩位一致
7. **灰度与回滚**：按版本/渠道/用户分层；保留 old contract 窗口

## 输出

- 变更说明（字段、默认值、兼容窗口）
- 验收用例（正常/异常/边界）
- 回滚开关与策略
