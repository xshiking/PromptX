# DB 层接入流程（MySQL 8.0 + Poolboy + mdb/logdb）

目标：把 DB 访问做成稳定可控的层，避免业务散落 SQL 与连接管理。

## 步骤

1. **确认归属**：业务数据走 mdb；日志/行为走 logdb（避免混用）
2. **定义接口**：以“领域方法”封装（不要让上层拼 SQL）
3. **Poolboy 配置**：大小、checkout 超时、worker 重连策略
4. **超时与重试**：统一 DB 超时；明确哪些操作可重试（幂等）
5. **事务边界**：尽量短；避免跨消息/跨进程保持事务
6. **慢查询治理**：SQL tag、调用点记录、索引策略

## 输出

- mdb/logdb 接口规范（方法列表 + 参数校验）
- 超时/重试/幂等等价表
- 慢查询排查与修复流程
