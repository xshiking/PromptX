# MySQL + Poolboy 使用模式（务实版）

目标：避免“连接池用错导致线上雪崩”，并把慢查询/事务边界治理前置。

## 关键约束

- Poolboy 是进程池，不是魔法：要定义 checkout 超时、worker 行为、错误恢复
- MySQL 8.0：关注超时、断线、死锁、慢查询、索引

## 推荐策略

- **超时统一**：网络超时、DB 超时、业务超时要一致（避免卡死进程）
- **断线重连**：worker 内部需能识别连接失效并重建
- **事务边界明确**：不要在 gen_server 状态里“挂着事务”跨多个消息
- **慢查询治理**：把“慢”变成指标（按 SQL tag/调用点聚合）

## 输出物

- DB 调用规范（超时、重试、幂等）
- mdb/logdb 分层边界（谁拼 SQL、谁负责连接）
- 慢查询排查清单（指标→采样→定位→索引/改写）
