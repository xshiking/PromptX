# 协议演进思维（XML + Proto 双格式）

目标：在不停服/弱停服的现实下，让协议可兼容、可灰度、可回滚。

## Proto 变更原则（建议）

- 优先“新增字段”，避免重用字段号
- 删除字段仅标记废弃，保留兼容窗口
- 对客户端不信任：服务端必须做缺省与边界校验

## XML + Proto 共存策略

- 明确“主协议/辅协议”的职责边界（别让两套都承担同样复杂度）
- 做好转换层（最好在 network 或协议层集中处理）
- 统一错误码与日志字段（不因格式不同而分裂）

## 灰度与回滚

- 版本号/feature flag：按客户端版本、渠道、用户分层控制
- 回滚策略：服务端优先支持 old contract 一段时间

## 输出物

- 协议变更 checklist（字段、兼容、生成、联调、回滚）
- module/method → protobuf message 映射表（单一真相源）
