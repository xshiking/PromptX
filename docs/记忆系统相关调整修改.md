# 记忆系统相关调整修改

本文档记录 PromptX 记忆系统（recall/remember）的近期调整与扩展。

---

## 1. recall 新增 scope 参数：支持项目级共享记忆

### 目标

- **共享**：允许团队把项目记忆放在项目目录下并共享（如提交到 Git）
- **安全**：`project` scope 读取时不创建/不写入数据库文件，避免污染仓库
- **兼容**：不传 `scope` 时保持原行为（读取个人记忆）

### scope 取值

| 值 | 说明 |
|----|------|
| `user` | 只读个人记忆（默认） |
| `project` | 只读项目记忆（共享） |
| `both` | 合并读取（个人 + 项目） |

> `remember` 行为不变：仍只写入 `user`（个人记忆）。

### 使用示例

**MCP：**
```json
{
  "role": "js-dev",
  "query": "pnpm",
  "scope": "project",
  "debug": true,
  "mode": "focused"
}
```

**CLI：**
```bash
promptx recall js-dev "pnpm" --scope=project --mode=focused
promptx recall js-dev "pnpm" --scope=both
```

### 存储路径约定

| scope | 路径 |
|-------|------|
| user | `~/.promptx/cognition/<roleId>/` |
| project | `<项目根>/.promptx/memory/cognition/<roleId>/` |

每个 roleId 目录下需包含：
- `network.json` - 认知网络图
- `engrams.db` - 记忆内容（SQLite）

---

## 2. recall 新增 debug 参数：MCP 输出诊断信息

当 `debug: true` 时，recall 的 MCP 返回内容中会附带 **Recall Diagnostics** 块，便于排查路径/文件是否存在：

- `basePath` / `roleDir`：解析到的目录
- `networkFilePath` / `engramsPath`：文件路径
- `networkExists` / `engramsExists`：文件是否存在

---

## 3. recall 的 mode 参数

| mode | 说明 |
|------|------|
| `balanced` | 平衡模式（默认）：精确与联想折中 |
| `focused` | 聚焦模式：更精确、常用记忆优先 |
| `creative` | 创造模式：广泛联想、远距离连接 |

---

## 4. 项目记忆热重载

当 project scope 的 `CognitionSystem` 在“文件尚未存在”时被缓存为空网络，后续若 `network.json` 被创建/同步到项目目录，**下一次 recall 会自动检测并热重载**，无需重启 MCP 服务。

---

## 5. 日志级别调整

为减少默认日志刷屏，以下定位日志已从 `info` 降级为 `debug`：

- `[CognitionManager] Resolved project cognition basePath`
- `[CognitionManager] Cognition storage probe`
- `[CognitionSystem] Skip memory load (readonly, db missing)` / `Loading memory (readonly)`

需要排查时，可设置环境变量 `LOG_LEVEL=debug`。
